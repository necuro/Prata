 Комплексное число состоит из двух частей — вещественной и мнимой. 
Один из способов записи такого числа выглядит как (3.0, 4.0). 
Здесь 3.0 —вещественная часть, а 4 .0 — мнимая. 
Предположим, что а = (А, Ві) и с = (С, Di). 
Нижепредставлены некоторые операции с комплексными числами:
• сложение: а + с = (А + С, (В + D)i)
• вычитание: а - с = (А - С, (В - D)i)
• умножение: а*с = (Ах С - ВхD, (AxD + В х С)і)
• умножение (х — вещественное число): ххс = (ххС, xxDi)
• сопряжение: ~а = (А, - Ві)
Определите класс complex так, чтобы следующая программа могла использовать его с корректными результатами:

using namespace std;
#include "complexO.h" //во избежание конфликта с complex.h

{
complex a (3.0, 4.0); // инициализация значением (3,4і)
complex c ;
cout << "Enter a complex number (q to quit) :\n";// Ввод комплексного числа (q для завершения)
while (cin » с)
{cout « "с is " « с « '\n'; // значение с
cout << "complex conjugate is " « ~c « ' \n';// значение сопряженного числа
cout << "a is " << a « '\n'; // значение аg
cout << "a + с is " << a + с << ' \n'; // значение а + с
cout << "a — с is " << a — с << ' \n'; // значение а — с
cout « "a * с is " << a * с << '\n'; // значение а * с

return 0;
}

He забывайте, что вы должны перегрузить операции << и >>. 
В стандарте C++ комплексных чисел — и намного более уже присутствует поддержка развитая,
чем в этом примере — в заголовочном файле complex, поэтому во избежание конфликтов назовите свой файл complexO.h.
